# Pages And UI Surface

This file inventories the current mobile UI. Route names use Expo Router names from `apps/mobile/app/**`; deep links use the app scheme shape `onewallet:///route` where applicable. Some route starts were unreliable from an already-mounted release activity during the 2026-05-30 release pass, so normal UI navigation remains the preferred visual QA path.

## Navigation Shell

| Surface        | Source                                        | Purpose                                                                                                                             | Notes                                                                                                   |
| -------------- | --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| Root stack     | `apps/mobile/app/_layout.tsx`                 | Auth provider, ledger provider, theme provider, stack screens, persistence lifecycle, native notification sync, Android widget sync | All screens are headerless except modal behavior configured at stack level.                             |
| Bottom tabs    | `apps/mobile/app/(tabs)/_layout.tsx`          | Home, Transactions, Calendar, Planner, Accounts                                                                                     | Uses a bottom-positioned material top-tab navigator with lazy screens and normal-flow bottom safe area. |
| Drawer         | `apps/mobile/src/components/AppDrawer.tsx`    | Global route shortcuts and sign out                                                                                                 | Sections: Daily, Money, Planning, Loans & credit, Tools, footer Settings/Sign out.                      |
| App back layer | `apps/mobile/src/components/AppBackLayer.tsx` | Custom Android back handling for overlays and route fallbacks                                                                       | Predictive/system back is covered in QA run `BACK-001`.                                                 |
| Not found      | `apps/mobile/app/+not-found.tsx`              | Unknown-route fallback                                                                                                              | Keep as framework entrypoint even with low references.                                                  |

## Navigation Notes

- Drawer primary routes: Home, Transactions, Add record, Review.
- Drawer money routes: Accounts, Reports, Categories, Cards, Currencies.
- Drawer planning routes: Planner, Calendar, Planned payments.
- Drawer credit routes: Loans, Loan forecast.
- Drawer tool routes: Auto Capture, Imports, Rules, Wallet snapshot, Notifications.
- Footer routes: Settings and Sign out.
- `/add` uses transparent modal presentation with custom source animation behavior.
- `/account/new`, `/budgets/new`, and `/goals/new` use bottom modal presentation.
- Back behavior should dismiss visible app layers before popping routes. Root back exits to launcher on Android.
